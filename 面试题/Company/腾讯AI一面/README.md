# 1.  请解释浏览器中DOM和BOM的含义，以及它们之间的联系和区别
 > DOM: 文档对象模型, 将 html 文档看成是一个对象, dom 的顶层是 document 对象, 让 js 可以操作 html
 > BOM: 浏览器对象模型, 将浏览器看成一个对象, bom 的顶层是 window 对象, 让 js 操作浏览器窗口

# 2.  请描述http状态码301和302的区别，如何理解它们的应用场景和对搜索引擎的优化  
 > 301: 永久重定向, 搜索引擎会将旧的 url 替换为新的 url, 并将旧的 url 对应的流量全部迁移到新的 url 上
 > 302: 临时重定向, 搜索引擎会将旧的 url 替换为新的 url, 但不会将旧的 url 对应的流量全部迁移到新的 url 上, 搜索引擎会根据 http 响应头中的 location 字段来判断重定向的目标 url

# 5.  如何使用webWorker来提升前端性能，并举例说明适合使用webWorker的场景
 >  webWorker 是运行在后台的 js 线程, 不会阻塞主线程, 可以在后台处理一些耗时的同步的任务或者性能开销大的任务
 >  适合使用 webWorker 的场景:
 >  1.  计算密集型操作, 比如加密, 解密, 压缩等
 >  2.  网络请求, 比如 ajax 请求, 图片加载等
 >  3.  其他需要在后台处理的任务, 比如文件上传, 下载等

# 13.  link标签中rel='preload'和rel='prefetch'的作用和区别  
 > preload: 预加载, 浏览器会在页面加载完成后, 发起一个请求, 预加载资源, 等资源加载完成后, 再执行 js 代码
 > prefetch: 预获取, 浏览器会在页面加载完成后, 发起一个请求, 预获取资源, 等资源加载完成后, 等 js 代码执行到需要资源的位置时, 再执行 js 代码
